Build Process

7-step flow with parallel tracks — every module follows this exact sequence

STEP 1 DEFINE
Output: Problem statement, scope, API surface list, key constraints
Owner: Adit + Mukul  ·  Track: Single sequential step — no parallel work here
STEP 2 DESIGN — Two parallel tracks run simultaneously
TRACK A Frontend
A
Phase A: UX Design (Product + Dhruv + Ashish)
B
Phase B: Visual Spec lock — Satyam approves
C
Phase C: Copy + fixtures from API Contract
D
Phase D: Frontend Build
E
Phase E: Adit L2 review (15 rules)
F
Phase F: Sign-off
BLOCKER: Cannot build until API Contract exists
TRACK B Backend
A
Phase A: L1+L2 test cases (Mukul — BEFORE code)
L1 = does service return correct values per spec? L2 = does it follow architecture rules? Backend in isolation only — no user scenarios here.
B
Phase B: Code Generation (Devender)
Devender generates service code. Phase A must exist first.
C
Phase C: Spec-to-code review (Adit 10-point)
Code review against Build Spec — field names, types, state machine, FM-rules. Not a test run.
D
Phase D: L1+L2 test execution (Mukul runs Phase A cases)
Runs the L1+L2 cases written in Phase A against the service. Backend only — no frontend, no user flows. L3+L4 run later in Step 5.
E
Phase E: Fix loop
F
Phase F: Sign-off — backend service complete
Track B output = API Contract. Track A phases D–F (build onward) cannot start until contract exists.
CONVERGENCE POINT: API CONTRACT  ·  Track B backend provides contract. Track A frontend consumes it.
STEP 3 CONTRACT
Output: API Contract v1.x locked  ·  Convergence of Track A design work + Track B backend
STEP 4 MODULE TEST STRATEGY — L3 + L4 + Canonical Fixtures
Why this is separate from Track B Phase A: L1+L2 test cases (backend service behavior) can be written before code exists. L3+L4 require the API Contract to exist first — you can't write integration or user scenarios without knowing the contract shape.

L3 — Integration cases (15–25): Frontend + backend together. Contract honoured end-to-end. Run at E2.
L4 — User scenarios (5–10): Real user flows on the full module. Run at E3 on a real device.
Canonical fixture set: Derived from API Contract. Product creates these. Frontend QA cannot run without them.

Owner: Mukul writes (L3+L4), Product creates fixtures — Adit reviews  ·  Blocker: E2 and E3 cannot start until this exists
STEP 5 EXECUTE — 4 stages, each blocks the next
E1
Developer self-check
Before merge. Dev validates own code against spec checklist.
→
E2
Integration — runs L3 cases
After merge. Runs L3 integration cases from Step 4. 100% pass required before E3. Step 4 must exist first.
→
E3
Manual QA — runs L4 cases
Real device, real Android. Product runs L4 user scenarios from Step 4 against canonical fixtures. Both must exist before E3 starts.
→
E4
Field test — real CSPs
Real CSPs on real network. Blocks ship. No defined test cases — observational.
STEP 6 GATES — All 5 must pass
G1 Truth gate
G2 Architecture gate
G3 Integration gate
G4 User gate
G5 Performance gate
STEP 7 SHIP + REGRESSION PACK
Output: Deployed module + regression pack (10–15 scenarios — blocks future releases if missing)
Staged rollout: Wiom staff (3–5 days) → 20–50 CSPs (1–2 weeks) → All CSPs

Amendment Decision Tree
Something is wrong or missing → follow this tree before touching any code or spec
Q1: Does the spec describe the correct behavior?
YES → Code is wrong CODE_BUG Fix code, no spec change needed
NO → Q2: Does the OS/Architecture cover this scenario?
YES → Spec missed it SPEC_GAP Fix spec, code follows
NO → Q3: Is this a value/parameter only?
YES → PARAMETER_CHANGE Update config only
NO → Behavior undefined everywhere DESIGN_GAP STOP — escalate to Satyam
HOTFIX Active production harm → emergency track. Bypass normal flow only with Satyam approval.
Frontend Sub-types
RENDER_BUG
Wrong color, wrong badge rendering — fix in code, no spec change
COPY_BUG
Wrong Hindi text displayed — fix copy, check against contract
ARCHITECTURE_DRIFT
Computing locally what must be computed by backend. Always PR rejection — not "fix next sprint"
PERFORMANCE_REGRESSION
G5 gate failure — must be resolved before ship
VISUAL_SPEC_GAP
Spec is missing a rendering rule → STOP. Do not invent rendering. Escalate.
CONTRACT_MISMATCH
Gateway returning wrong shape → backend fixes. Frontend does NOT workaround.
10 Non-Negotiables
01Mukul writes test plan BEFORE Devender codes — no exceptions
02Canonical fixtures from API Contract BEFORE frontend builds — truth must exist first
03Track A and Track B converge at API Contract — neither can skip it
04L2 architecture review on every frontend PR — Adit's 15-rule checklist
05E2 must reach 100% before E3 starts — integration must be clean
06E4 field test before any module ships — real CSPs, real devices
07Regression pack created at ship — blocks future releases if missing
08Amendment type determined by spec/OS — not by person's preference or convenience
09ARCHITECTURE_DRIFT = PR rejection — not "fix next sprint". Hard stop.
10VISUAL_SPEC_GAP = STOP — do not invent rendering. Escalate to Satyam.
